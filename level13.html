Unlocking Data with Keys

In Level 11, we used lists to store items in order. But what if you want to store a piece of information and give it a specific name? In Level 13, we introduce **Dictionaries**. At **Flora World Coding**, we think of a dictionary like a real-life dictionary: you look up a "word" (the Key) to find its "definition" (the Value). This is the most efficient way to store related data, like a user's profile information or a character's stats in a game.

Dictionaries use curly braces `{ }` and consist of pairs. Each pair has a key followed by a colon and then the value, like this: "name": "Flora". Unlike lists, dictionaries don't use numbers to find items. Instead, you use the Key. If you have a dictionary called player, typing player["health"] will instantly give you the value stored under that key. This "mapping" makes your code much more readable and organized.

Dictionaries are used everywhere in the modern web. When an app like Spotify stores a song's title, artist, and duration, it uses a dictionary-like structure called JSON. By mastering dictionaries, you are learning how to handle structured data, which is a key requirement for getting approved by Google AdSense and becoming a professional software engineer. It allows you to create complex systems that feel intuitive and organized.

plant = {
  "type": "Sunflower",
  "height": 150,
  "health": "Happy"
}
